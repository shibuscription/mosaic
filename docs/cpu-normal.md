# Normal CPU 解析ドキュメント

## この CPU の位置づけ

Normal は、MOSAIC の標準的な中位 CPU です。  
現行コード上では Easy と同じ評価プロファイル型で、`NORMAL_PROFILE` により重みだけを引き締めた派生ではありますが、実際の対局感はかなり変わります。

- Easy より「危険を嫌う」度合いが強い
- SOPHIA ほど相手最善返しを具体的には読まない
- 初中級者にとって、考えている感じはあるが理不尽すぎない相手を狙っている

## 一言でいうと

Normal は、「Easy と同じ見方で全合法手を採点するが、得失の差をもう少しまじめに受け取り、揺らぎを減らした CPU」です。

## 基本的な手選びの流れ

手順自体は Easy と同じです。

1. `getLegalMoves` で全合法手を列挙する
2. 各手を `placeManualPiece` で1手進めた局面に変換する
3. `evaluateMoveCore` で採点する
4. 小さめのノイズを足す
5. 上位候補の中から重み付きで選ぶ

違いは、何を見るかではなく、各評価軸の重みと候補の絞り方です。  
そのため実装としては近縁でも、打ち筋は「Easy の上位互換」ではなく、「同じ目でより慎重に選ぶ版」に見えます。

## 候補手の集め方

Normal も全合法手を見ます。  
危険手を先に切るルールベースのフィルタはありません。

- 候補生成は全合法手
- 事前除外なし
- 1手先の結果局面を全件評価

この点は Easy と同じですが、Normal は後段の評価差とランダム性の弱さによって、結果として候補をかなり絞ります。

## 評価していること

Normal が見ている軸は Easy と同一です。  
ただし、`NORMAL_PROFILE` では多くの値が強めになっており、「最低限の危険回避」「最低限の好手選択」が実際に見えるようになります。

### 1. 自動積み上がりの利得

`autoCountWeight`、`autoSelfWeight`、`autoLevelWeight` が Easy より高くなっています。  
つまり Normal は、

- 連鎖が起きること自体
- それが自分色であること
- それが高い段であること

を、Easy より真面目に評価します。

人間向けに言えば、「ただ置ける手」より「盤面を実際に進める手」を選びやすいです。

### 2. 相手に与える利得の嫌い方

`autoEnemyPenalty` が Easy より強く、相手色の自動積み上がりを伴う手をより嫌います。  
Easy でもここは見ていますが、Normal では「今得だが相手にも得」の評価がやや厳しくなります。

この差が、プレイヤーから見た「Easy より不用心さが減った」印象につながります。

### 3. 局所形の評価

`estimateLocalShapeBonus` の

- 3個同色形
- 2個同色形

への加点も Normal の方が強めです。  
単に置けるだけではなく、「その一手が次の形作りになっているか」を少し強く見ます。

ただし、ここで見ているのはあくまで着手近傍の最大4つの 2x2 です。  
盤全体の構図を読むわけではありません。

### 4. 露骨なブロック

`blockBonus` も強くなっています。  
1段上に置くことで直下の相手候補をふさぐ手を、Easy より積極的に拾います。

これは「最低限の危険回避」の中でも分かりやすい部分です。  
人間が見て「そこは止めるよね」という場所を拾う頻度が上がります。

### 5. 即時脅威の増減

`threatReduceWeight` が強く、`threatIncreasePenalty` もより重くなっています。  
Normal は、相手の即時脅威数を減らす手をちゃんと好み、増やす手をかなり嫌います。

このため、Easy に比べると

- 相手にすぐ3化されそうな形を放置しにくい
- 自分の一手で危険窓を広げる手が減る

という差が出ます。

### 6. 終盤価値

`endgameWinWeight` と `endgameThreatPenalty` も Easy より強いです。  
終盤に入ると、

- 自分の即勝ち筋
- 相手の即勝ち筋

の差をより重く扱うため、対局終盤は Easy より現実的になります。

## Easy との差分を人間向けに言うと

Normal は Easy と同じ景色を見ています。  
違うのは、その景色の受け止め方です。

- Easy は「良さそう」を広く拾う
- Normal は「危なそう」をやや強く嫌う
- Easy は揺らぎで別候補に流れやすい
- Normal は評価差をそのまま選択に反映しやすい

そのため、アルゴリズムの骨格は近くても、プレイヤーからは「Easy はたまに甘いが、Normal は最低限ちゃんとしている」と見えます。

## あえて見ていないこと

Normal は Easy より慎重ですが、SOPHIA のような具体的な返し読みはありません。

- 相手の全合法返しを並べて最大打点を計算しない
- 受け渡しリスクを一手ごとに解析しない
- 自分の手が将来の自爆予約になっているかを個別にチェックしない
- 連鎖の深さや相手の深い発火筋を分解しない

言い換えると、「よくある危険」は見るが、「強者が狙う具体的な罠」はまだ読んでいません。

## 局面ごとの振る舞い

### 序盤

- 中央と上段への軽い志向は Easy と同じ
- 形作り加点がやや効く
- まだ脅威数差が大きくなりにくい

序盤は「少し筋が良い Easy」に見えます。

### 中盤

- 自動積み上がりの価値が強く効く
- 相手脅威を増やす手を嫌う
- ブロックボーナスが目立つ

中盤で Normal らしさが最も出ます。  
プレイヤーから見ると、「露骨な危険は避けながら、取れる利得はちゃんと取る」ように見えます。

### 終盤

- `endgameBonus` により、勝ち筋と負け筋の即時性をより意識する
- ただし深い詰み読みではなく、あくまで即勝ち手の数ベース

そのため終盤はそこそこ締まりますが、SOPHIA ほど冷徹ではありません。

## ランダム性と揺らぎ

Normal もランダム性は残しますが、Easy よりかなり狭いです。

- ノイズは `noiseRange = 1.8`
- 最善との差許容は `nearBestMargin = 10`
- 候補プールは最大 `poolLimit = 3`

つまり Normal は、

- 評価の近い3手程度で揺れることはある
- でも Easy のように5手近くまで広くぶれにくい

という設計です。

このため、同じ局面で完全固定にはならない一方、プレイヤーには「それなりに一貫した CPU」に見えます。

## 人間向けに言い換えると

Normal は、人間でいえば次のような思考です。

- 全候補に一度は目を通す
- すぐ得する手を拾う
- 近場の形が伸びるかを見る
- 相手に明らかな好形を渡す手はできれば避ける
- 最後は上位数手の中から少し揺らして選ぶ

「露骨な悪手は減らしたいが、深読みで時間をかけるほどではない初中級者」寄りの棋風です。

## なぜ“ほどよい相手”に見えるのか

Normal がほどよく見える理由は、見ているものと見ていないもののバランスです。

- 目先の得はちゃんと拾う
- 露骨な危険もそこそこ避ける
- でも相手最善返しまでは読まない
- 少しランダムなので毎回最善機械にはならない

このため、プレイヤーは

- Easy より手応えはある
- でも理不尽に読み負けさせられる感じは薄い

と受け取りやすくなります。

## 強み

- Easy より実戦的で、明らかな危険手を減らせる
- 連鎖、妨害、即時脅威の3つを最低限バランスよく見ている
- ランダム性が抑えられているため、勝敗が「なんとなく」になりにくい
- 初中級者の学習相手として、勝っても負けても理由を感じやすい

## 弱み

- 相手最善返しを具体的に読まないので、受け渡しの質に弱い
- 局所形評価が中心で、全体構想は浅い
- 「この手は今は普通だが、次に大事故になる」という局面を見落とす
- 終盤も即勝ち手の数ベースなので、読み合いの精度には限界がある

## なぜ不思議な手を選ぶことがあるのか

Normal の不思議な手は、だいたい次の競合で生まれます。

- 自分の連鎖利得は高い
- だが相手脅威も少し増える
- 近場の形は良くなる
- しかも中央や上段補正も乗る

Normal はこの競合を一つの足し算で処理しているため、人間が「その危険は重い」と感じる局面でも、総合点で押し切られることがあります。

## 今後の改善余地

- 相手の最善返しを1手だけ軽く見る処理を追加し、SOPHIA 未満の中間層を厚くする
- 即時脅威の個数だけでなく、脅威の質も少し区別する
- 終盤ではランダム性をもう少し絞り、勝ち切り性能を上げる
- Easy と Normal の差を、重み差だけでなく評価項目差でも明確化する
