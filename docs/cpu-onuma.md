# Onuma CPU

## 概要

`Onuma` は、MOSAIC 開発者の大沼さん案をベースにした実験用 CPU です。  
固定の初期比重を土台にしつつ、盤面条件に応じて加点減点して 1 手先の局所評価で着手を選びます。

初版では深い探索や連鎖読みは入れず、まずは「動く」「見える」「係数をいじれる」を優先します。

## 目的

- 固定比重 + 条件加点減点型の CPU を、軽量な比較実験用として追加する
- 実戦テストしながら係数調整しやすい形にする
- `SOPHIA` や `kobalab CPU` とは別の発想の比較対象を増やす

## 基本方針

- 139 マスすべてに初期比重を持つ
- 合法手ごとに初期比重を起点に score を組み立てる
- 盤面条件は順番に加点減点して合算する
- まずは 1 手先だけを見る
- 最終 score の最高値から tolerance 範囲内の候補を抽出し、その中からランダム選択する

## 盤面による比重変動ルール

初版では次を順番に適用します。

1. `base weight`
   - 段ごとの固定初期比重
2. `adjacent bonus`
   - 自コマのタテヨコナナメ隣接 1 個ごとに `+3`
3. `opponent threat penalty`
   - その手の周辺で、相手があと 1 手でボーナスを取れる地点 1 件ごとに `-5`
4. `self-only bonus penalty`
   - 自コマだけで四角形を作って自分のボーナスになる場合 `-3`
5. `deny opponent bonus reward`
   - 置くことで相手のボーナス候補を 1 件防げたら `+5`
6. `allow opponent bonus penalty`
   - 置いた結果として相手のボーナスが発生したら 1 件ごとに `-10`
7. `mixed bonus reward`
   - 中立コマまたは相手コマ 1 つを含む形で自分のボーナスを獲得できたら `+7`

複数四角形に関与する場合は合算します。  
四角形判定は既存ルール実装の `evaluateSquareForAutoPlacement` と整合させます。

## 難易度の考え方

元案では「最高 score からどこまで許容するか」で難易度を表現します。

- easy 相当: `highest - toleranceEasy`
- normal 相当: `highest - toleranceNormal`
- hard 相当: `highest - toleranceHard`

現在の Web 版では、この評価系を一般向けの `Easy / Normal / Hard` に割り当てます。  
通常 UI に `Onuma` 名は出さず、難易度は CPU 選択側で決まります。

## 実装方針

- 実装本体は `src/game/onuma.ts` に分離する
- 初期比重は docs に明記しつつ、実装側では静的テーブルを持つ
- 合法手生成は既存 `getLegalMoves` を使う
- 実着手は `placeManualPiece` を使って既存ルールと揃える
- debug 表示用に、候補ごとの内訳を構造化して返す
- terminal 局面では安全な debug state を返し、白画面を防ぐ
