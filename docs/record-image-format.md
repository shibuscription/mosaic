# 棋譜画像形式

## この文書の目的

この文書は、現行 Web 版が出力する棋譜シート PNG の構成を整理し、他実装でも同じ思想で再現しやすくすることを目的とします。  
UI のスクリーンショット説明ではなく、出力仕様の説明として記述します。

## 棋譜画像の用途

棋譜画像は、主に人間向けの閲覧と共有のためのフォーマットです。

- 対局内容を一覧しやすくする
- 盤面の各段をまとめて見返せるようにする
- `.mosaic` のような再生用データとは別に、静的な共有物として扱う

## 画像全体の構成

現行 Web 版の PNG は、おおむね次の構成を取ります。

- 上部にタイトル
- その下にメタ情報
  - 対戦モード
  - 勝者
  - 総手数
  - 出力日時
- その下に各段の盤面を縦に並べた本体
- 背景は白基調

## 各盤面の並べ方

各段の盤面は、上から下へ次の順で縦積みされます。

- `level 6` の 1x1
- `level 5` の 2x2
- `level 4` の 3x3
- `level 3` の 4x4
- `level 2` の 5x5
- `level 1` の 6x6
- `level 0` の 7x7

つまり、最上段から最下段へ向かって、盤が大きくなる方向で配置されます。  
各盤は中央揃えで描画され、盤ごとの間には一定の余白を置きます。

## 段数ごとの見せ方

- 各段は独立した正方グリッドとして描画します
- 各セルは空欄でも枠線を持ちます
- 各段のサイズ差がそのまま盤面構造の理解につながることを重視します

この表現の狙いは、実際の 3D 見た目を模写することではなく、「どの段にどの手番番号が入ったか」を読みやすくすることです。

## 色の意味

文字色は先手・後手を区別するために使います。

- 先手側の手番番号: 黒系
- 後手側の手番番号: 赤系

ここでいう先手・後手は、`openingTurn` を基準に判定します。  
盤面テーマ色とは独立した表現です。

## 注記や括弧表現がある場合の意味

- 通常の手置きは `23` のように手数だけを表示します
- 自動積み上がりは `(23)` のように括弧付きで表示します

この括弧表現により、「その位置がプレイヤーの直接入力ではなく、その手に付随して発生した自動配置である」ことを区別します。

## 見やすさのためのルール

- 背景は白、セル内は淡い色、枠線は控えめな線色にする
- 情報量は手数と先後区別に絞る
- 段のサイズ差が自然に読み取れるよう、各段を等間隔で縦配置する
- 盤面テーマや 2D 対局 UI の装飾を持ち込みすぎない

## 実装依存になりすぎない説明

現行 Web 版では Canvas を用いて PNG を生成していますが、本質は次の点です。

- 各段を個別グリッドで表現すること
- 手置きと自動配置を表記で区別すること
- 先後を色で区別すること
- 対局メタ情報を同じ画像内に収めること

したがって、他実装では PDF や SVG にしても、上記の読み取り規則が保たれていれば互換的な発想といえます。

## 将来拡張の余地

- 各段のラベル表示
- 着手者名やテーマ名の付与
- PDF や印刷向け出力への拡張
- 連鎖段数や注釈欄の追加
