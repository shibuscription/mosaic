# Easy CPU 解析ドキュメント

## この CPU の位置づけ

Easy は、MOSAIC の入門向け CPU です。  
現行コード上では `src/game/cpu.ts` の `EASY_PROFILE` を使う評価プロファイル型 CPU で、4種類の中では最も判断を簡略化しています。

- 想定相手は、ルールを覚えたい人、連鎖の基本だけ掴みたい人、まず勝つ体験を作りたい人です
- Normal より危険回避が浅く、SOPHIA のような相手最善返しの比較はしません
- kobalab CPU のような別思想の探索型でもありません

弱い版ではなく、意図的に「考えすぎない」設計にしてあります。

## 一言でいうと

Easy は、「全合法手をざっと採点し、その場で得しそうな手を広めに拾い、少しランダムに選ぶ CPU」です。

## 基本的な手選びの流れ

現行実装の流れはかなり素直です。

1. `getLegalMoves` で全合法手を列挙する
2. 各手について `placeManualPiece` を実行し、その手で起こる自動積み上がりまで含めた結果局面を作る
3. `evaluateMoveCore` で点数化する
4. `scoreMove` で小さなランダムノイズを足す
5. `pickMoveWithVariance` で上位候補の中から重み付きで1手を選ぶ

ここで重要なのは、Easy でも完全ランダムではなく、全合法手を一度は見ていることです。  
ただし、見る深さは「自分がその手を打った直後」までで止まります。

## 候補手の集め方

Easy は危険手を事前に除外しません。  
入口ではあくまで全合法手を同列に集め、あとから軽い評価で並べ替えます。

- 全合法手を見る
- 危険手を先に落とすフィルタはない
- 注目局面だけ抜き出す前処理もない
- 探索木を伸ばすこともしない

このため、見た目には「まじめに全部見ている」のに、結果はどこか素朴な打ち方になります。  
理由は、候補生成ではなく評価の簡略さにあります。

## 評価していること

Easy の評価式は `evaluateMoveCore` が中心です。  
人間向けに訳すと、次の要素を足し引きしています。

### 1. その手で今すぐどれだけ得するか

`placeManualPiece` 後の `lastAutoPlacements` を見て、次を評価します。

- 自動積み上がりが何個起きたか
- そのうち自分色のものが何個あるか
- 相手色の自動積み上がりが混ざったか
- 高い段で起きたか

つまり Easy は、「連鎖が起きた」という事実そのものにはかなり素直に点を入れます。  
ただし、その連鎖が将来どこまで続くかまでは読んでいません。

### 2. 近くの 2x2 の形が良くなるか

`estimateLocalShapeBonus` では、着手位置の周囲にある最大4つの 2x2 を見て、

- 自色が 3 個になる形
- 自色が 2 個になる形

に加点します。

人間の感覚でいうと、「この一手で近場の形が少し良く見えるか」を見ています。  
局所形しか見ていないので、離れた場所の構想や全体の受け渡しはここでは拾いません。

### 3. 露骨な相手の上積みをその場で止めるか

`estimateBlockBonus` は、1段上に打つことで、直下にある相手色の自動積み上がり候補をふさぐ手にボーナスを出します。

これはかなり限定的な妨害評価です。

- 相手が今まさに上に乗れそうな場所をふさぐ
- ただし見るのは直下1か所だけ
- もっと複雑な危険連鎖は見ない

つまり Easy は、「分かりやすい即席の邪魔」はできますが、複合的な守りにはなりません。

### 4. 上段と中央を少し好む

`move.level * levelWeight` と `centerPreference` により、

- 高い段に打つ手
- その段の中央に近い手

を少し好みます。

これは厳密な読みではなく、MOSAIC で価値が出やすい場所に軽く寄せるための誘導です。  
入門向け CPU として、盤面の価値が高そうな場所へ自然に寄せる働きがあります。

### 5. 相手の即時脅威が増えるか減るか

`countImmediateThreats` は、各 2x2 を見て

- 相手色が3個ある
- 相手色2個 + 中立1個がある

ような「すぐ危なそうな形」の数を数えます。  
Easy は着手前後でその数がどう変わるかを比べ、

- 減らせたら加点
- 増やしたら減点

します。

ここで見ているのはあくまで「即時脅威の個数」です。  
脅威の質や、相手がどの順で使うかまでは見ません。

### 6. 終盤で勝ちに近いか

`endgameBonus` は、両者の残り駒がある閾値以下に入った終盤で効き始めます。

- その局面がすでに自分勝ちなら加点
- 相手より自分の残り駒が少ないなら少し加点
- 自分に即勝ち手が何個あるかを加点
- 相手に即勝ち手が何個あるかを減点

Easy でも終盤は多少締まりますが、ここでも見るのは「即勝ちがあるか」までで、深い詰み読みではありません。

## あえて見ていないこと

Easy がやさしく見える理由は、見ていることより、見ていないことにあります。

- 相手の最善応手を個別にシミュレートしない
- 2手先、3手先の連鎖の伸びを比較しない
- 候補手同士の危険差を細かく分解しない
- 自分の手が将来の自爆予約になっているかを詳しく見ない
- 連鎖の深さ、相手への受け渡し量、返しの最大値を評価しない

そのため Easy は、

- 目先で少し得なら採りやすい
- その得が次の相手の大得につながるかは見落としやすい
- 局所形がきれいだと気に入りやすい

という性格になります。

## 局面ごとの振る舞い

Easy に明確な「序盤用ロジック」「中盤用ロジック」はありません。  
ただし、評価項目の効き方の結果として、局面ごとに次のように見えます。

### 序盤

- 中央寄りを少し好む
- 近くの形を作る手を拾いやすい
- まだ脅威数が少ないので、守りの差が出にくい

結果として、「なんとなく形の良さそうな場所」に寄る素朴な序盤になります。

### 中盤

- 自動積み上がりの点が効きやすくなる
- 相手脅威の増減も少し効く
- ただし、返しの最大打点は見ない

このため、中盤は「今得するなら行く」傾向が強くなります。  
人間から見ると、勢いはあるが不用心な手が混ざるのはここです。

### 終盤

- `endgameBonus` が乗り始める
- 即勝ち筋や相手即勝ち筋の個数を少し見る
- それでも深い詰みの読み合いはしない

結果として、終盤だけ少し現実的になりますが、SOPHIA ほどの締まり方にはなりません。

## ランダム性と揺らぎ

Easy のランダム性はかなりはっきり入っています。

### ノイズ

`scoreMove` で、各手のスコアに `noiseRange = 3.2` のノイズを加えます。  
これは4 CPUの中でも強めです。

### 候補プール

`pickMoveWithVariance` では、

- 最善手との差が `nearBestMargin = 18` 以内
- その中から最大 `poolLimit = 5`

というかなり広い候補群を残します。

### 重み付き抽選

完全等確率ではなく、点が高い手ほど当たりやすい重み付き抽選です。  
それでも Easy は上位5候補程度まで平気で揺れるので、同一局面でも打ち筋がぶれやすいです。

## 人間向けに言い換えると

Easy は、人間にたとえると次のような打ち方です。

- まず合法手を一通り眺める
- すぐ得する手や、近くの形が良くなる手を気に入る
- 露骨な危険は少し嫌う
- ただし、次に相手からどれだけ強く返されるかは深く読まない
- 最後は「このへんが良さそう」で少し気分も混ぜて選ぶ

「ちゃんと考えてはいるが、怖い変化には踏み込まない入門者寄りの棋風」と言えます。

## 強み

- 完全ランダムではないので、MOSAIC の基本形を学ぶ相手として機能する
- その場の連鎖や局所形には反応する
- 盤の中央や上段に自然に寄るため、初心者相手でもそれらしい対局になる
- ランダム性があるので、同じ局面ばかり繰り返しにくい

## 弱み

- 相手の最善返しを見ないため、受け渡しに非常に弱い
- 大きな危険が「脅威数」ではなく「脅威の質」にある局面を苦手とする
- 局所形ボーナスに引っ張られて、全体では危ない手を選ぶことがある
- 終盤でも即勝ちの数しか見ないため、詰めの精度は高くない

## なぜ不思議な手を選ぶことがあるのか

Easy の不思議な手は、たいてい次のどれかです。

- その場で自動積み上がり点が高い
- 局所 2x2 の形がよく見える
- 中央・上段ボーナスが乗る
- 近い点数の候補の中からランダムに選ばれた

人間は「次の返しが怖い」と感じる局面でも、Easy はそこを明示的に比較していないため、結果だけ見ると妙手ではなく「不用心な好形手」に見えます。

## 今後の改善余地

- 入門向けの性格を保ったまま、極端な自爆だけ減らす軽い返しチェックを入れる
- 序盤は今のまま、中終盤だけ危険手を少し狭くする
- ランダム性は残しつつ、相手即勝ちを渡す手だけは候補プール外にする
- 初心者体験を壊さない範囲で、「やさしいが理不尽ではない」線を微調整する
