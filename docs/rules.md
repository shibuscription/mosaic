# MOSAIC ルール仕様

## この文書の目的

この文書は、MOSAIC のルールを実装担当や AI が誤解しにくい形で整理することを目的とします。  
現行 Web 版の実装を基準にしつつ、別実装でも流用しやすいように、UI 表現よりもルールそのものを優先して記述します。

## 用語定義

- 盤面
  - 段ごとに大きさが異なる正方格子を積み上げた全体構造です。実装上は `board[level][row][col]` で表されます。
- 配置ポイント
  - 駒を置ける個々のマスです。各段に存在し、空であることと支持条件を満たすことが必要です。
- 駒
  - `blue`、`yellow`、`neutral` のいずれかの色を持つ要素です。中立駒は初期配置専用です。
- 段
  - 下から上へ向かう階層です。`level 0` が最下段、`level 6` が最上段です。
- 中央駒
  - ゲーム開始時に `level 0` の中央に置かれている中立駒です。
- 手番
  - どちらのプレイヤーが次に手置きするかを表す状態です。初期手番は `blue` です。

## 盤面構造の考え方

盤面は 7 段構造です。

- `level 0`: 7x7
- `level 1`: 6x6
- `level 2`: 5x5
- `level 3`: 4x4
- `level 4`: 3x3
- `level 5`: 2x2
- `level 6`: 1x1

上の段の 1 マスは、ひとつ下の段の 2x2 に支えられているとみなします。  
このため、上段に置けるかどうかは、下段 4 マスの充足で決まります。

## ゲーム開始時の状態

- `blue` と `yellow` はそれぞれ 70 個の持ち駒を持ちます。
- 盤面は空ですが、`level 0` の中央 `(3,3)` のみ中立駒が 1 個置かれています。
- 初手は `blue` です。
- 勝者は未確定です。

## 各手番でプレイヤーが行うこと

各手番でプレイヤーが行う本質的な処理は次の 3 段階です。

1. 自分の色の駒を 1 個、合法な配置ポイントに手置きする
2. その結果発生する自動積み上がりを、候補がなくなるまで解決する
3. 勝敗判定を行い、未終局なら手番を交代する

## 駒の配置ルール

手置きできる条件は次のとおりです。

- 対象の配置ポイントが盤面内にある
- 対象の配置ポイントが空である
- `level 0` では、空であれば置ける
- `level 1` 以上では、直下の 2x2
  - `(row,col)`
  - `(row+1,col)`
  - `(row,col+1)`
  - `(row+1,col+1)`
  の 4 マスがすべて埋まっている必要がある

この条件を満たさない手は非合法手です。

## 積み上がりが発生する条件

同じ段の任意の 2x2 について、4 マスすべてが埋まると、その真上にあたる 1 マスに自動配置の候補が生じます。

色判定は次のとおりです。

- 4 マス中 `blue` が 3 個以上なら `blue` の自動配置候補になる
- 4 マス中 `yellow` が 3 個以上なら `yellow` の自動配置候補になる
- 中立駒はどちらの色にも数えない
- どちらも 3 個未満なら自動配置は発生しない

## 積み上がり後に何が起きるか

自動配置候補が選ばれると、その位置に該当色の駒が置かれます。

- 自動配置された駒も、そのプレイヤーの持ち駒を 1 個消費します
- 対象プレイヤーの残り駒が 0 の場合、その候補は実行されません
- 実行された自動配置は、新たな 2x2 完成を生み、さらに上位の候補を誘発することがあります

## 連鎖の扱い

自動配置候補は、候補がなくなるまで連続して処理されます。  
複数候補が同時に存在する場合、現行実装では次の固定順で解決されます。

1. 段の小さい順
2. 行の小さい順
3. 列の小さい順

この順序は、見た目の演出ではなく、実装上の結果を一意にするための重要な仕様です。

## 合法手・非合法手の考え方

合法手は、現在の盤面と支持条件に基づいて `canPlaceAt` が真になる手です。  
非合法手は主に次のいずれかです。

- 盤面外への配置
- すでに埋まっている位置への配置
- 上段で、直下 2x2 の支持がそろっていない位置への配置
- すでに終局している局面での追加操作
- 自分の残り駒が 0 の状態での手置き

## ゲーム終了条件

ゲームは、いずれかのプレイヤーの残り駒が 0 以下になった時点で終局します。  
判定は手置きと連鎖解決が終わったあとに行われます。

## 勝敗判定

- `blue` の残り駒が 0 以下なら `blue` の勝ち
- `yellow` の残り駒が 0 以下なら `yellow` の勝ち
- 同一手の処理で両者とも 0 以下になった場合は、その手を着手した側が勝ち

現行実装では引き分けはありません。

## 実装上の補足

### Web 版での表現

- 現行 Web 版では、段を半マスずつずらした 2D 表示と、終局後の 3D 表示を持ちます。
- ただし、これらはあくまで視覚表現であり、ルールの本質は「段別グリッド」「支持条件」「2x2 からの自動積み上がり」にあります。
- 連鎖演出や CHAIN 表示は視認性のための UI であり、解決順そのものはルール側の固定順に従います。

### 別実装でも守るべき本質的ルール

- 盤面サイズが `7,6,5,4,3,2,1` と縮む段構造であること
- 上段配置が下段 2x2 の支持に依存すること
- 2x2 完成時に 3 個以上同色なら真上へ自動配置すること
- 自動配置でも持ち駒を消費すること
- 候補が複数あるときの解決順が固定であること
- 勝敗が残り駒の 0 到達で決まること

- 中立駒を初期中央以外に拡張する可能性を持たせるか

- BGA 版や他実装で、表示上の演出とルール処理の境界をどこまで共通化するか
